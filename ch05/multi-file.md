# 終わりに: まだUzumibiではできないこと

## 複数ファイルのアプリケーション記述

現在のUzumibiでは、Rubyのアプリケーションコードは **単一ファイル** （`lib/app.rb` や `lib/consumer.rb`）に記述する必要があります。

通常のRuby開発では、`require` や `require_relative` を使ってコードを複数ファイルに分割し、モジュール化するのが一般的です。しかし、Uzumibiではmruby/edgeの制約により、これらのファイル読み込み機能は利用できません。

### 制約の背景

mruby/edgeでは、Rubyソースコードはビルド時にmrubyバイトコード（`.mrb`）にコンパイルされ、WASMモジュールに静的に埋め込まれます。

```rust
// wasm-app/build.rs
let code = include_str!("../lib/app.rb");
// このコードを mrbc でコンパイルし、.mrb ファイルを生成
```

このプロセスでは1つのRubyファイルのみがコンパイル対象となるため、`require` による外部ファイルの読み込みは現時点ではサポートされていません。

### 現在の回避策

大規模なアプリケーションを記述する場合、以下の方法で対応できます。

1. **1ファイルに全コードを記述**: クラス定義やヘルパーメソッドをすべて `lib/app.rb` 内に記述します。mruby/edgeは通常のRubyと同様にクラス定義や多くの言語機能をサポートしているため、1ファイル内でもある程度の構造化は可能です。

2. **ビルドスクリプトでファイルを結合**: `wasm-app/build.rs` を改造して、複数のRubyファイルを1つに結合してからコンパイルすることも技術的には可能です。ただし、公式にサポートされている方法ではありません。

### 今後の対応

複数ファイルのサポートは、Uzumibiの今後の開発で優先的に取り組む予定の機能です。mruby/edgeのIREP（内部表現）レベルでの複数バイトコードのリンク機能が実現すれば、より自然なファイル分割が可能になるでしょう。
